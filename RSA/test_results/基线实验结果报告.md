### 实验一：RSA 攻击与防御（Pollard’s rho）——结果总结报告

#### 1. 实验目的

验证 **模数大小** 与 **素数均衡性** 对 RSA 因数分解抵抗力的影响；用 Pollard’s rho 作为通用型分解算法，分别观察**小模数可攻破**与**大模数有效防御**，以及**失衡模数的脆弱性**。

------

#### 2. 测试配置与指标

- 配置：`baseline`（均衡 p、q）与 `weak_imbalanced`（极不均衡）
- 位数：64、80、96、112、128、256、512（另含 256 weak_imbalanced）
- 每组：多次随机试验
- 主要指标：**ASR（Attack Success Rate）**、**平均攻击时间**（s）

------

#### 3. 关键结果（来自实验输出）

##### 3.1 baseline（均衡 p、q）

| 位数 | ASR      | 平均攻击时间（s） | 说明                                  |
| ---- | -------- | ----------------- | ------------------------------------- |
| 64   | **100%** | 0.027             | 毫秒级至十毫秒级即可分解              |
| 80   | **100%** | 0.458             | < 1s，仍极易分解                      |
| 96   | **100%** | 22.519            | 十余秒级，仍可稳定分解                |
| 112  | **80%**  | 617.496           | 出现失败样本；成功案例均值约 219.669s |
| 128  | **0%**   | 2502.970          | 在限时/步数内基本失败                 |
| 256  | **0%**   | 4656.116          | 失败为主，耗时数千秒                  |
| 512  | **0%**   | 9539.208          | 失败为主，耗时上万秒量级              |

**结论（baseline）：**
 出现明显“拐点”在 **112→128 位**之间：112 位仍可部分成功但耗时剧增；到了 128 位，在你的实现与参数上 **ASR≈0**。当位数进一步增大（256、512），Pollard’s rho 基本失效（在你的运行上限内）。

------

##### 3.2 weak_imbalanced（256 位，极不均衡）

| 配置                     | ASR      | 平均攻击时间（s） | 说明             |
| ------------------------ | -------- | ----------------- | ---------------- |
| 256（p≈40 位，q≈216 位） | **100%** | **4.845**         | 全部成功，且很快 |

**结论（失衡）：**
 即使 **n = 256 位**，若 **p 很小**、**q 很大**，则 **极易被分解**。失衡模数是致命弱点。

------

#### 4. 与预期对比（你原始设计的“预期结果”）

- 你预期：**256–512 位**可能还有 **10–50%** 的 ASR；
- 实测：**256 / 512 baseline 的 ASR = 0%**（在你的最大步数/时间限制内）。

> 解释：Pollard’s rho 的期望复杂度随素因子位长急剧上升；在你当前实现与参数设置（最大迭代、尝试次数、单机环境）下，**到 128 位已近乎不可行**，更高位自然更难。你的实验很好地展示了 **“位数带来的安全性跃迁”**。

------

#### 5. 结论（核心要点）

1. **小模数可攻破**：64/80/96 位均可稳定分解；112 位出现边界效应（部分成功但耗时大）。
2. **128 位成为实测拐点**：在当前实现和参数下，ASR 降为 0%。
3. **大模数有效防御**：256 / 512 位 baseline 在实验限制下基本不可分解。
4. **失衡模数极其危险**：256 位但失衡（p≈40 位）时，ASR=100%，平均仅 ~4.8s 即被攻破。**总位数并不能掩盖小素因子的可分解性**。

> 一句话凝练：**要安全，既要“大”（位数足够），也要“匀”（p、q 均衡）。**
